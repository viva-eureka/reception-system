## 1. 管理画面へのアクセス

受付画面の右下にある **⚙ 管理** をタップ → 4桁の PIN を入力。

**初期 PIN は `1234` です。必ず最初に変更してください（→ [2-4. PIN の変更](#2-4-管理-pin-の変更)）。**

> PIN を忘れた場合は、システム管理者に連絡するか、Supabase ダッシュボードの `reception_settings` テーブルで `admin_pin` の値を確認してください。

### 管理画面のタブ構成

| タブ | 主な用途 |
|-----|---------|
| ダッシュボード | 本日の来訪者・直近の入退館記録 |
| 入退館記録 | 来訪履歴の検索・CSV 出力・手動退館処理 |
| 招待管理 | 招待状の作成・URL 共有・一覧確認 |
| 会議室 | 会議室の空き状況・本日のスケジュール |
| 設定 | 会社情報・通知・受付ボタン・PIN 変更・自動退館 |

## 2. 初期設定

管理画面 → **設定** タブで以下を順番に設定してください。

### 2-1. 会社情報

| 項目 | 内容 |
|-----|------|
| 会社名 | 来訪者向けページに表示されます |
| 会社ロゴ | 招待状ページに表示（240×120px 推奨、PNG / JPG） |
| 住所・電話番号 | 招待状ページに表示 |

### 2-2. Google Chat Webhook URL

> ℹ️ **管理者が初期セットアップ時に1回設定すれば全員共通で使えます。** すでに設定済みの場合はスキップしてください。

来訪通知や招待状作成の通知を受け取るスペースの Webhook URL を設定します。

1. Google Chat → 通知を受け取りたいスペース → **設定（⚙）→ Apps & integrations → Webhooks**
2. **Add Webhook** → 名前を入力 → URL をコピー
3. 管理画面 → 設定 → 通知設定 → **Webhook URL** に貼り付けて保存

> 通知先スペース: **テスト用受付管理**（本番移行時は現行の「受付管理システム」スペースに変更）

### 2-3. 自動退館設定

チェックイン後に一定時間が経過すると自動的に退館処理します。

| 項目 | デフォルト | 説明 |
|-----|-----------|------|
| 一般来訪者 | 120分 | 商談・打ち合わせなど |
| 配送業者 | 5分 | 荷物の受け渡しのみ |

> 管理画面を表示している間、1分ごとに自動チェックされます。

### 2-4. 管理 PIN の変更

**デフォルトの `1234` から必ず変更してください。**

管理画面 → 設定 → **管理 PIN** → 新しい4桁の数字を入力して保存。

### 2-5. 受付ボタンのカスタマイズ

iPad の受付メニューに表示するボタンを編集・並び替えできます。  
管理画面 → 設定 → **受付ボタン管理**

## 3. Google Apps Script の個人設定（カレンダー連携）

**Google カレンダーで外部ゲストを招待したとき自動で招待状を作成する機能は、カレンダーを使う人それぞれがセットアップする必要があります。**

> システム管理者がすでに1件セットアップしている場合も、各メンバーが自分のアカウントで設定しなければ自分のカレンダーから招待状は作成されません。

### 3-1. プロジェクトの作成

1. [script.google.com](https://script.google.com) にアクセス（自分の Google アカウントでログイン）
2. **新しいプロジェクト** をクリック
3. [reception-calendar.gs](https://github.com/viva-eureka/reception-system/blob/main/gas/reception-calendar.gs) を開き、内容をまるごとコピーして貼り付け
4. **保存**（画面上部のフロッピーディスクアイコン、または `Ctrl+S` / `⌘+S`）
5. 左メニュー **サービス（＋）** → **Google Calendar API** を追加
6. スクリプト上部の定数を確認（変更不要のはずですが念のり確認）

```javascript
const RECEPTION_URL  = "https://reception-eureka.com";
const COMPANY_DOMAIN = "viva-eureka.co.jp";
```

### 3-2. シークレットの設定（1回だけ）

1. スクリプト内の `setSecretOnce()` 関数を探す
2. `"ここに入力"` の部分を、管理者から共有された `INVITATION_SECRET` の値に書き換える
3. 上部メニュー → **実行** → `setSecretOnce` を選択して**1回だけ実行**
4. 実行後、`"ここに入力"` の文字列を削除して保存（コードにシークレットを残さない）

### 3-3. トリガーの登録（1回だけ）

`setupTrigger()` を**1回だけ手動実行**します。

| トリガー | 動作 | 各自の設定 |
|---------|------|----------|
| `onCalendarEventUpdated` | 自分のカレンダーで外部ゲスト招待を作成・更新したとき | **必須**（各自） |
| `syncRoomAvailability` | 15分ごとに会議室の空き状況を同期 | 誰か1人でOK |

> `onCalendarEventUpdated` は **自分のカレンダーの変更にしか反応しません**。カレンダーで招待を送る予定がある人は必ず設定してください。  
> `syncRoomAvailability` は誰か1人が設定すれば全員分の会議室データが更新されます。

### 3-4. 動作確認

1. Google カレンダーで、社外のメールアドレス（Gmail など）を含む予定を作成
2. 数分後に Google Chat の通知スペースに通知が届けば成功
3. ゲストのメールアドレスに QR コード付きの案内メールも届きます

> **うまくいかない場合**: Apps Script の **実行数（ログ）** を確認してください。  
> `script.google.com` → プロジェクト → **実行数** でエラー内容が確認できます。

## 4. 日常の操作ガイド

### 来訪者の受付

来訪者は iPad で以下のいずれかの方法で受付できます：

| 方法 | 操作 |
|-----|------|
| QR コード | 招待状の QR コードをかざす |
| 名刺撮影 | 名刺をカメラにかざすと氏名・会社名を自動読み取り |
| キーボード入力 | 名前・会社名をタッチキーボードで入力 |
| 声で受付 | マイクに向かってお名前を話しかける |

### 招待状の作成

**カレンダー経由（推奨）**: 社外ゲストを含む予定を Google カレンダーで作成すると自動生成。

**手動作成**: 管理画面 → **招待管理** → 来訪者名・担当者・日時を入力 → 発行された URL を来訪者に送付。

### 退館処理

以下のいずれかの方法で退館処理されます：

| 方法 | タイミング |
|-----|----------|
| 自動退館 | チェックイン後に設定時間が経過したとき（管理画面を開いている間） |
| 手動退館 | 管理画面 → 入退館記録 → 該当行の **退館にする** ボタン |

### 来訪ログ・監査ログの確認

管理画面サイドバー下部 → **🔍 来訪・監査ログ** をクリック。

| タブ | 内容 |
|-----|------|
| 来訪ログ | 来訪者一覧・在館状況・CSV ダウンロード |
| 監査ログ | 管理ログイン・設定変更・招待作成などの操作履歴 |

## 5. 動作確認チェックリスト

初期設定が完了したら、以下を確認してください。

### 設定確認
- [ ] 管理 PIN を `1234` から変更した
- [ ] 会社名・ロゴが正しく設定されている
- [ ] Google Chat Webhook URL が設定されている（管理者確認でも可）
- [ ] 営業時間が正しく設定されている
- [ ] 自動退館の時間設定を確認した

### 招待状フロー
- [ ] 管理画面 → 招待管理 → テスト招待を作成
- [ ] 発行された URL にアクセスし、QR コードが表示される
- [ ] URL を来訪者に送付する方法を確認した（LINE / メール）

### カレンダー連携（GAS 設定済みの場合）
- [ ] 外部ゲストを含む予定を作成して通知が届くか確認
- [ ] ゲストにメールが届くか確認

### 受付フロー（iPad で確認）
- [ ] QR コードをかざすと受付完了画面が表示される
- [ ] 名刺撮影で氏名・会社名が読み取れる
- [ ] Google Chat にスタッフ通知が届く

### ログ確認
- [ ] 来訪・監査ログ画面が開ける
- [ ] 入退館記録に来訪が記録されている
