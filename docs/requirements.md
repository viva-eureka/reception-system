## 1. プロジェクト概要

### 背景

現在、受付管理に「ラクネコ」（月額課金のiPadネイティブアプリ）を使用している。
月額コストの削減と、招待・カレンダー登録・当日受付の煩雑さを解消するため、自社開発に切り替える。

### 目的

| # | 目的 |
|---|---|
| 1 | ラクネコの月額費用を削減する |
| 2 | 招待作成〜来訪当日までのフローを自社業務に最適化する |
| 3 | Googleカレンダー・Google Chat との連携を自動化する |
| 4 | 将来的な機能拡張の自由度を確保する |

### スコープ（フェーズ1）

- 受付端末（iPadキオスク） — PWA（Webアプリ）
- 管理コンソール（PC/iPad ブラウザ）
- Google Chat 通知
- 入退館記録

## 2. 機能要件

### 2-1. 受付端末（キオスク画面）

| ID | 機能 | 詳細 | 優先度 |
|---|---|---|---|
| F-01 | 待ち受け画面 | ロゴ・時計・日付・あいさつ文を表示。タップで受付開始 | Must |
| F-02 | 営業時間外表示 | 設定時間外は「ただいま営業時間外です」を表示し操作不可 | Must |
| F-03 | 受付メニュー | 「配送業者」「ご予約・招待の方」の2択 | Must |
| F-04 | QR受付 | カメラでQRコードを読み取り、招待情報と照合して受付完了 | Must |
| F-05 | 名刺撮影受付 | カメラで名刺を撮影、OCRで氏名・会社名を抽出し確認・修正して受付 | Should |
| F-06 | キーボード受付 | 氏名・会社名をテキスト入力して受付 | Must |
| F-07 | 配送業者ワンボタン | ボタン1回で配送受付完了・通知送信 | Must |
| F-08 | 受付完了画面 | あいさつ文を表示後、一定秒数で待ち受けに自動復帰 | Must |
| F-09 | 背景テーマ | 5種類のテーマを設定画面から選択可能 | Nice |

### 2-2. 招待管理

| ID | 機能 | 詳細 | 優先度 |
|---|---|---|---|
| F-10 | 招待作成 | 来訪者名・会社・担当者・日時・会議室を登録 | Must |
| F-11 | QRコード生成 | 招待ごとに一意のQRコードを生成 | Must |
| F-12 | 招待メール送信 | QRコード添付の招待メールを来訪者に送信 | Must |
| F-13 | カレンダー連携 | Google Calendar に予定を自動作成し、会議室リソースを予約 | Must |
| F-14 | 招待一覧 | 作成済み招待の一覧表示・削除 | Must |
| F-15 | 受付済みフラグ | QR受付完了後に「受付済」表示 | Must |

### 2-3. 入退館管理

| ID | 機能 | 詳細 | 優先度 |
|---|---|---|---|
| F-16 | 入館記録（自動） | 受付完了時に自動で入館記録を作成（氏名・会社・種別・担当・日時） | Must |
| F-17 | 退館記録（手動） | 管理画面から任意のタイミングで退館記録を追加 | Must |
| F-18 | 記録一覧 | 時系列での入退館ログ表示、在館状態の判定 | Must |
| F-19 | 退館記録（自動）| 来訪者QR退館対応（フェーズ2） | Won't |

### 2-4. 通知

| ID | 機能 | 詳細 | 優先度 |
|---|---|---|---|
| F-20 | 来訪通知 | 受付完了時に Google Chat Webhook へ通知（氏名・会社・担当・会議室・時刻） | Must |
| F-21 | 配送通知 | 配送受付時に Google Chat Webhook へ通知 | Must |
| F-22 | 退館通知 | 退館記録時に Google Chat Webhook へ通知 | Should |

### 2-5. 設定・管理

| ID | 機能 | 詳細 | 優先度 |
|---|---|---|---|
| F-23 | 会社情報設定 | 会社名・タグラインを変更 | Must |
| F-24 | 受付時間設定 | 開始・終了時刻を設定 | Must |
| F-25 | 会議室管理 | 会議室名の追加・削除 | Must |
| F-26 | Webhook設定 | Google Chat Webhook URL を設定 | Must |
| F-27 | プライバシー設定 | 受付完了画面の表示モード（氏名のみ／会社名+氏名／汎用文）を選択 | Must |
| F-28 | テーマ設定 | 待ち受け背景テーマを選択 | Nice |

## 3. 非機能要件

### 3-1. ユーザビリティ

| # | 要件 |
|---|---|
| U-01 | 受付端末は来訪者が**初見で迷わず操作できる**こと（ガイドテキスト、大きなボタン） |
| U-02 | 受付完了まで**3タップ以内**を目標とする |
| U-03 | iPadのソフトキーボード表示時にレイアウトが崩れないこと |
| U-04 | 管理コンソールはPC（1280px以上）とiPad（768px以上）の両方で使えること |

### 3-2. パフォーマンス

| # | 要件 |
|---|---|
| P-01 | 受付完了まで**体感3秒以内**（ネットワーク遅延を除く） |
| P-02 | 初回ロード（iPad Safari）で**5秒以内**に操作可能になること |

### 3-3. 可用性・信頼性

| # | 要件 |
|---|---|
| V-01 | 受付時間内の**稼働率99%以上**を目指す |
| V-02 | Wi-Fi 一時切断時でも**待ち受け画面の表示は維持**されること（Service Worker / キャッシュ） |
| V-03 | Google Chat 通知失敗時は**リトライ or エラーログ**を残すこと |

### 3-4. セキュリティ

| # | 要件 |
|---|---|
| S-01 | 管理コンソールは**認証必須**（Supabase Auth）。受付端末は認証不要 |
| S-02 | QRコードのトークンは**予測不可能な一意の値**を使用する |
| S-03 | 受付完了画面への**会社名表示はデフォルト無効**（プライバシー設定 F-27） |
| S-04 | 来訪者の個人情報（氏名・会社名・来館履歴）は **Supabase の Row Level Security（RLS）**で保護する |
| S-05 | 保存データの**保持期間を設定**し、期限超過データは自動削除または匿名化する |

### 3-5. 保守性

| # | 要件 |
|---|---|
| M-01 | フロントエンドとバックエンドを明確に分離し、片方の変更が他方に影響しないこと |
| M-02 | コードは GitHub で管理し、`main` マージで自動デプロイ（GitHub Pages / Vercel）されること |
| M-03 | 設定（受付時間・テーマ等）はコード変更なくUI から変更できること |

## 4. システム構成（案）

```
┌─────────────────────────────────────────────┐
│                  クライアント                     │
│                                               │
│  受付端末 (iPad)          管理コンソール (PC)    │
│  PWA / Safari             ブラウザ              │
│                                               │
└──────────────┬──────────────────┬────────────┘
               │                  │
               ▼                  ▼
┌─────────────────────────────────────────────┐
│              Next.js (Vercel)                 │
│  受付画面 / 管理画面 / API Routes             │
└──────────────┬──────────────────┬────────────┘
               │                  │
       ┌───────┘                  └──────────┐
       ▼                                     ▼
┌─────────────┐                   ┌──────────────────┐
│  Supabase   │                   │  外部サービス      │
│  - Postgres │                   │  - Google Chat    │
│  - Auth     │                   │  - Google Calendar│
│  - Storage  │                   │  - OCR (Vision)   │
│  - Realtime │                   │  - メール送信      │
└─────────────┘                   └──────────────────┘
```

### 採用技術（案）

| 役割 | 技術 | 理由 |
|---|---|---|
| フロントエンド | Next.js + React + Tailwind CSS | 管理画面と受付端末を同一コードベースで管理 |
| バックエンド / DB | Supabase | Auth・RLS・Realtime が揃っている。月$25〜 |
| ホスティング | Vercel | GitHub連携で自動デプロイ。無料枠あり |
| 認証 | Supabase Auth | 管理画面のみ使用 |
| メール送信 | Resend | 日本語メール・添付対応。月3000通まで無料 |
| カレンダー連携 | Google Calendar API（サービスアカウント） | 会議室リソース予約に管理者権限が必要 |
| 名刺OCR | Google Cloud Vision API | 日本語名刺の認識精度が高い |
| 通知 | Google Chat Incoming Webhook | すでに利用中 |

## 5. 開発フェーズ

### フェーズ1（優先実装）

受付端末の基本動作と管理コンソールをSupabaseで永続化し、実運用可能な状態にする。

| # | タスク |
|---|---|
| 1 | Supabase プロジェクト作成・テーブル設計（invitations / logs / settings） |
| 2 | 招待管理のCRUD実装（Supabase連携） |
| 3 | 入退館記録の永続化 |
| 4 | Google Chat Webhook への実通知 |
| 5 | 管理コンソールの認証（Supabase Auth） |
| 6 | Vercel へのデプロイ・自動デプロイ設定 |

**目標**: ラクネコの「受付端末」「記録」「Chat通知」をすべて代替できる状態

### フェーズ2（次期実装）

| # | タスク |
|---|---|
| 1 | Google Calendar 連携（招待作成→予定・会議室自動登録） |
| 2 | 招待メール実送信（Resend + QR添付） |
| 3 | 名刺OCR実装（Google Cloud Vision API） |
| 4 | PWA対応（Service Worker / オフラインキャッシュ） |

### フェーズ3（将来検討）

- 来訪者自己登録フォーム
- 退館QRスキャン
- 来訪分析・レポート
- 複数拠点対応
