## 受付端末シナリオ（来訪者側）

### US-01 招待済み来訪者がウェルカム表示からQRで受付する

**アクター**: A1  
**事前条件**: 招待メールが届いており、QRコードを画面または印刷で手元に持っている  
**前提**: 予約開始15分前から待ち受け画面に「お打ち合わせ予約 ○○様」のウェルカムボタンが表示されている

1. 来訪者が受付端末の前に立つ
2. 待ち受け画面に自分の名前が入ったウェルカムボタンを見つけてタップ
   - または「画面をタップ」→「ご予約・招待の方」→「QRコードをかざす」→QR読み取り
3. 受付完了画面に氏名（設定によっては会社名も）と「担当者がまいります」が表示
4. Google Chat に担当者へ通知が飛ぶ
5. 入館記録が自動作成される

**成功条件**: 担当者が通知を受け取り来訪者を迎えに来る  
**例外**: QRが読み取れない → キーボード入力に誘導

---

### US-02 来訪者が名刺を撮影して受付する

**アクター**: A1 / A2  
**事前条件**: 名刺を持参している  
**方式**: OCRなし。撮影画像をGoogle Chatに自動送信し担当者が目視確認する（iPadインカメラでのOCR精度不足のため）

1. 受付メニューで「ご予約・招待の方」→「名刺を撮影」を選択
2. カメラガイド枠に名刺を合わせて「撮影する」をタップ
3. 撮影した名刺画像がGoogle Chatに自動送信される
4. 氏名・会社名をキーボードで入力（名刺を見ながら入力）
5. 「受付する」をタップ
6. 受付完了 → 担当者へ通知（名刺画像付き）

**担当者側**: Chat通知に名刺画像が添付されるため、来訪者の会社・役職等をすぐ確認できる

2. 氏名（必須）・会社名（任意）を入力 →「受付する」
3. 受付完了 → 担当者へ通知

### US-04 配送業者がワンボタンで受付する

**アクター**: A3

2. 受付完了画面が表示される（5秒で待ち受けに自動復帰）
3. Google Chat に「配送業者が到着」通知、配送記録が自動作成される

### US-05 営業時間外に来訪者が訪問する（受付不可）

**アクター**: A1 / A2 / A3

1. 受付時間外に端末を操作
2. 「ただいま営業時間外です」と受付時間が表示される
3. 操作不可（受付フローに進めない）

### US-06 営業時間外に来訪者が訪問する（受付可能・設定ON時）

**アクター**: A1 / A2 / A3  
**設定**: 「時間外でも受付を許可する」ON  

1. 「ただいま営業時間外です」と受付時間が表示されるが「画面をタップして受付を開始」も表示
2. タップすると通常の受付フローに遷移
3. 受付完了 → 担当者へ通知（時間外である旨を通知に付記する）

## 管理・運用シナリオ

### US-07 受付から来客対応完了までの一連フロー（現行 → 新システム）

**アクター**: A1/A2, A4, A7  
**概要**: 現行オペレーションを踏襲しつつ、通知・対応者記録・担当者連絡を自動化する

#### 現行フロー（ラクネコ）
1. 来訪者が受付 → ラクネコがGoogle Chat共有スペースに通知
2. 手の空いている事務員（A7）が通知に気づき、Chatに「私が対応します」と**手書き**で書き込む
3. A7がロビーへお出迎えに行く
4. A7が来訪者を会議室へ通す
5. A7が担当者（A4）に連絡（声がけ・Chat・電話など状況により**手動**）
6. 会議終了後、会議室を片付ける

#### 新システムでのフロー
1. 来訪者が受付 → システムが以下を**同時に**送信：
   - **共有スペース**（事務員全員）: インタラクティブカード通知
   - **担当者（A4）のDM**: 「○○様がお見えです」直接通知
2. 事務員（A7）がカードの **「✋ 対応する」ボタン**を押す
   - カードが即座に「**[A7名]が対応中（HH:MM）**」に書き換わり二重対応を防止
   - 誰がいつ対応したかが記録される
3. A7がロビーへお出迎えに行く ← **変わらず**
4. A7が来訪者を会議室へ通す ← **変わらず**
5. 担当者（A4）はDM通知の **「🚶 今向かいます」「⏰ 少々お待ちを」** ボタンで状況を返信
   - A7が担当者に個別連絡する**手間が省ける**
6. 会議終了予定時刻に自動リマインダー通知（会議室片付け促進）

**改善効果**:
- ステップ2: Chat手書き → ボタン1押し＋自動記録
- ステップ5: 手動連絡 → DM通知で自動化（最大の時間短縮）
- 対応者・対応時刻が記録として残る

### US-08 受付管理者が来訪予約を作成し招待を送る

**アクター**: A5  

1. 管理画面 → 招待管理 →「新規招待を作成」
2. 来訪者名（必須）・会社/所属（**任意**）・メールアドレス（任意）・担当者・日時を入力
3. 会議室選択欄に**空き/使用中がリアルタイム表示**される → 空いている部屋を選択
4. 「招待を作成」→ QRコードが自動生成・Googleカレンダーに予定と会議室予約が自動作成
5. 送付方法を選ぶ：
   - **① メールで送る** → QR添付メールを自動送信
   - **② メッセージをコピー** → 受付コード・会議室・案内文を含む文章がクリップボードにコピーされる。LINEやMessengerに貼り付けて送付
   - **後で送る** → 一覧に保存しておき後から送付

**改善点（ラクネコとの差）**:
- 会社名が必須でなくなり、フリーランス・個人の来訪者にも対応
- 会議室予約が被っているかどうかを選ぶ前に確認できる（「確定後に弾かれる」が解消）
- メール以外のチャネル（LINE・Messenger等）でも招待を送れる

### US-15 受付管理者が会議室の空き状況を確認する

**アクター**: A5 / A4 / 全社員  
**用途**: 来客招待時だけでなく、社内会議の会議室探しにも使える  

1. 管理画面 → 会議室 タブを開く
2. 各会議室の「今すぐ使えます / 使用中」が一覧で分かる
3. 終日タイムラインで今日の予約状況を確認
4. 現在時刻の縦線で「今どこが空いているか」が視覚的に把握できる

**特記**: Google Calendar の room resource（リソース）カレンダーのみ参照。各社員の個人カレンダーは一切参照しない。

---

### US-09 来訪者が来社しなかった（ノーショー）

**アクター**: A4 / A5  
**トリガー**: 予約開始時刻から設定時間（例: 15分）が経過しても受付未完了の招待が存在する  

1. システムが未受付招待を検知し、担当者（A4）へGoogle Chat自動通知を送信
   - 通知例:「山田様（株式会社サンプル商事）14:00のお約束ですが、まだ受付されていません。来社されていますか？」
2. 担当者が応答
   - **来社している場合**: 担当者が管理画面から手動で受付処理
   - **来ていない場合** または **一定時間応答なし**: システムが招待をノーショーとしてマークし、会議室予約を自動開放
3. 管理画面 → 招待管理 にノーショー記録が残る

---

### US-10 受付したが退館処理をしていない

**アクター**: A5  
**トリガー**: 入館記録はあるが退館記録がないまま一定時間が経過した  

1. 設定した時刻（例: 23:59）になると、在館中の来訪者全員に自動退館記録を作成する（**自動退館設定で確定**）
2. 管理画面 → 入退館記録 で「退館（自動）」として記録される
3. 手動で退館させたい場合は随時「退館にする」ボタンから操作可能

**将来拡張（フェーズ3検討）**: 会議室の予約終了時間と照合して、担当者に「○○様は退館されましたか？」の確認通知を送る機能を追加予定

### US-11 受付管理者が入退館記録を確認・エクスポートする

**アクター**: A5 / A6  

1. 管理画面 → 入退館記録 を開く
2. 日付・担当者・区分（来訪/配送）でフィルタして絞り込む
3. 在館状態の来訪者を把握する
4. 「CSVダウンロード」で記録をエクスポート（監査対応）

---

### US-12 受付管理者が設定を変更する

**アクター**: A5  

変更できる設定一覧:
- 待ち受けテーマ（5種類）
- **受付時間（曜日別）** — 月〜日ごとに開始・終了時刻を個別設定。定休日はチェックをオフ
- **時間外でも受付を許可する**（ON/OFF）
- 受付完了画面の表示モード（プライバシー設定）
- **受付ボタン管理** — iPad受付メニューのボタンを追加・削除・並び替え（名前・絵文字・カラー・種別を設定）
- 会議室の追加・削除
- Google Chat Webhook URL

### US-13 受付管理者が入退館記録をCSVでエクスポートする

**アクター**: A5 / A6  
**事前条件**: 入退館記録が1件以上存在する  

1. 管理画面 → 入退館記録 を開く
2. 「☰ 表示項目」から出力したい列（会社・区分・動作・担当・人数・メモ等）をON/OFFで選択
3. 「↓ CSVダウンロード」をクリック
4. BOM付きUTF-8のCSVファイルがダウンロードされる（Excelでそのまま開ける）

**成功条件**: 表示中の列設定が反映されたCSVが正しく生成される  
**用途**: 監査対応・月次集計・外部報告

---

### US-14 Googleカレンダー拡張機能から招待QRを発行する（フェーズ1）

**アクター**: A4 / A5  
**事前条件**: Chrome/Edge に「ラクネコ（自社受付）」拡張機能がインストール済み  

1. Google Calendar で来客の予定を作成する
2. 予定編集画面に表示される「QR発行」ボタンをクリック
3. 来訪者の氏名・会社名・担当者を入力して発行
4. QRコードが生成され、招待メールを送信できる状態になる

> **NOTE**: フェーズ1はブラウザ拡張機能方式。管理者権限（サービスアカウント・委任設定）が不要で即導入可能。フェーズ2ではAPI連携に切り替えて完全自動化する予定。
