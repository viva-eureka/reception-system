## 管理画面の使い方

### 管理画面へのアクセス

受付画面の右下にある **⚙ 管理** ボタンをタップ → 4桁の PIN を入力。

> PIN を忘れた場合は、Supabase ダッシュボード → `reception_settings` テーブルの `admin_pin` キーの値を確認してください。

### 管理画面のタブ構成

| タブ | 主な用途 |
|-----|---------|
| ダッシュボード | 本日の来訪者・直近の入退館記録 |
| 入退館記録 | 過去の来訪履歴の検索・確認 |
| 招待管理 | 招待状の作成・URL 共有・一覧確認 |
| 会議室 | 会議室の空き状況・本日のスケジュール |
| 設定 | 会社情報・通知・受付ボタン・PIN 変更 |

### 受付端末に戻る

管理画面の左サイドバー下部にある **🔒 受付端末に戻る** をクリック。

## 招待状の管理

### 招待状とは

来訪者が受付端末でスムーズにチェックインできるよう、事前に発行する URL（QR コード付き）です。

### 自動作成（Google カレンダー経由）【推奨】

1. Google カレンダーで来訪者（社外ゲスト）を招待した予定を作成
2. 自動的に招待状が作成され、Google Chat に通知が届く
3. Chat に届いた URL を来訪者に LINE・メール等で送付

> **ポイント**: 社外ゲストのメールアドレスが含まれている予定が対象です。社内のみの予定では招待状は作成されません。

### 手動作成（緊急時・カレンダー未登録の場合）

1. 管理画面 → **招待管理** タブ
2. 来訪者名・担当者名・日付・時間を入力して **招待状を作成**
3. 発行された URL をコピーして来訪者に送付

### 招待状 URL の共有方法

- **LINE**: URL をそのままテキストで送信
- **メール**: URL をメール本文に貼り付け
- **印刷**: URL にアクセスすると QR コードが表示されるので、来訪者のスマートフォンで保存

### 来訪者のチェックイン方法

来訪者が受付端末（iPad）で以下のいずれかを選択：

| 方法 | 操作 |
|-----|------|
| **QR コード** | 招待状の QR コードをカメラにかざす |
| **キーボード入力** | お名前・会社名をタッチキーボードで入力 |
| **声で受付** | マイクに向かってお名前を話しかける |

## 会議室ダッシュボード

### 空き状況の見方

- 🟢 **空き**: 現在予約なし
- 🔴 **使用中**: 現在予約あり（予約名・時間が表示）

### タイムライン

- 縦棒（青）= 現在時刻
- ピンクのブロック = 予約あり
- ホバーすると予約名・時間・予約者が表示

### データの更新タイミング

会議室データは **約15分ごと** に Google カレンダーと自動同期されます。  
最終同期時刻は画面右上に表示されています。

緊急に最新データを取得したい場合は、Apps Script プロジェクトで `syncRoomAvailability` を手動実行してください。

## よくある質問（Q&A）

### 招待状について

**Q. Google Chat に通知が来ない**  
A. 以下を確認してください：
1. 管理画面 → 設定 → Google Chat Webhook URL が正しく設定されているか
2. カレンダーの予定に**社外ゲスト**（社内ドメイン以外のメールアドレス）が含まれているか
3. Apps Script プロジェクトの実行ログ（Google Apps Script → 実行数）でエラーが出ていないか

**Q. 招待 URL を開くと 404 エラーが出る**  
A. Vercel の最新デプロイが完了していない可能性があります。Vercel ダッシュボードでデプロイ状況を確認してください。

**Q. 同じ人の招待状が複数作られてしまった**  
A. カレンダーで同じ予定を複数回更新・保存した場合に発生することがあります。管理画面 → 招待管理 から古い招待状を確認し、来訪者には最新の URL を案内してください。

**Q. 招待状に会社名が表示されない**  
A. Google カレンダーのゲストに個人メールアドレス（Gmail など）を使用している場合、会社名の自動推測ができません。手動作成から会社名を入力してください。

---

### 会議室について

**Q. 会議室のスケジュールが表示されない・古い**  
A. Apps Script の時間ベーストリガー（15分ごと）が正常に動作しているか確認してください。  
Google Apps Script → プロジェクト → 実行数 を開くと定期実行の履歴が確認できます。

**Q. 予約者名が表示されない**  
A. Apps Script の `syncRoomAvailability` 関数に `creator` フィールドが含まれているか確認してください。  
含まれていない場合は、以下を各イベントオブジェクトに追加して再実行：
```javascript
creator: (ev.getCreators()[0] || "").split("@")[0],
```

**Q. 存在しない会議室が表示される**  
A. 管理画面 → 設定 → 会議室 から不要な会議室を削除できます。

---

### iPad・キオスク端末について

**Q. カメラが起動しない（QR が読めない）**  
A. Safari の設定でカメラアクセスが許可されているか確認してください：  
設定 → Safari → カメラ → 「確認する」または「許可」

**Q. 音声受付が反応しない**  
A. マイクのアクセス許可を確認してください：  
設定 → Safari → マイク → 「確認する」または「許可」  
また、周囲が静かな環境でテストしてください。

**Q. 画面が暗くなってしまう**  
A. 設定 → 画面表示と明るさ → 自動ロック → **しない** に設定してください。

**Q. 誰かが管理画面を開けたまま放置している**  
A. 管理画面の左下 **🔒 受付端末に戻る** で受付画面に戻れます。  
次回からは Guided Access を有効にして管理画面への遷移を制限することを推奨します。

---

### 管理・設定について

**Q. 管理 PIN を忘れてしまった**  
A. Supabase ダッシュボード → Table Editor → `reception_settings` →  
`key = 'admin_pin'` の行の `value` を確認または変更してください。

**Q. 会社ロゴを変更したい**  
A. 管理画面 → 設定 → 会社ロゴ から画像ファイルをアップロードできます。  
推奨サイズ: 横 240px × 縦 120px 以内、PNG または JPG

## トラブルシューティング

### 症状別の確認フロー

#### 「招待状が作成されない」

```
カレンダー予定に社外ゲストがいるか？
    ↓ いない → 対象外（仕様）
    ↓ いる
Apps Script の実行ログを確認
    ↓ エラーあり → エラー内容を確認
    ↓ HTTP 401 → SECRET / INVITATION_SECRET の設定を確認
    ↓ HTTP 500 → Supabase の接続・スキーマを確認
    ↓ 正常終了なのに通知が来ない → Webhook URL を確認
```

#### 「受付後に Chat 通知が来ない」

```
管理画面 → 設定 → Webhook URL が設定されているか確認
    ↓ 設定されていない → Webhook URL を設定
    ↓ 設定されている
Vercel → Functions ログ で /api/notify のエラーを確認
```

#### 「会議室データが更新されない」

```
Apps Script の時間トリガーが存在するか確認
    ↓ ない → syncRoomAvailability の15分トリガーを作成
    ↓ ある
実行ログを確認（HTTP レスポンスコードを確認）
    ↓ 401 → SECRET の設定を確認
    ↓ 200 → フロントエンドのキャッシュかもしれない → リロード
```

## Tips・活用術

### 来訪管理を効率化する

**Tip 1: カレンダー予定のタイトルを来訪目的にする**  
Google カレンダーの予定タイトルが招待状の「目的」として自動的に引き継がれます。  
例: `【外】株式会社○○ 商談` のように書くと来訪者ページに表示されます。

**Tip 2: 会議室を予定に追加する**  
カレンダー予定に会議室（リソースカレンダー）を追加すると、招待状に会議室名が自動的に記載されます。  
来訪者は受付後に「○○室へお越しください」と案内されます。

**Tip 3: ゲストの表示名を正しく設定する**  
Google カレンダーのゲストに「表示名」がある場合はその名前が使われます。  
名刺交換後に Contacts に登録しておくと、次回から自動的に正しい名前で招待状が作られます。

---

### 会議室の活用

**Tip 4: タイムラインの現在時刻マーカーを活用**  
会議室ダッシュボードの青い縦線が現在時刻です。次の予約まであとどれくらいか一目でわかります。

**Tip 5: ホバーで詳細確認**  
タイムラインのブロックにマウスを乗せると「予約名・時間・予約者」が表示されます。  
iPad では長押しで確認できます。

---

### 運用のコツ

**Tip 6: 定期的に招待管理を確認**  
過去の招待状が溜まっていくと一覧が見づらくなります。月1回程度、古い招待状を整理することをお勧めします。

**Tip 7: テスト招待で動作確認**  
新しいスタッフが入社した際など、テスト用の招待状を作成して一連の流れを確認しておくと安心です。

**Tip 8: Google Chat のスペースを活用**  
受付通知用の専用 Google Chat スペースを作成し、そこに Webhook を設定すると、  
来訪通知がまとめて確認でき、見逃しが減ります。

---

### セキュリティのベストプラクティス

**Tip 9: 管理 PIN は定期的に変更する**  
3〜6ヶ月に1度、管理 PIN を変更することを推奨します。

**Tip 10: INVITATION_SECRET はランダムな文字列を使う**  
`my-company-2024` のような推測しやすい値は避け、パスワードマネージャーで生成したランダムな文字列を使用してください。

**Tip 11: スタッフの退職時は Webhook 設定を見直す**  
個人の Google Chat Webhook を登録している場合、退職したスタッフの設定が残ったままになっていないか確認してください。
